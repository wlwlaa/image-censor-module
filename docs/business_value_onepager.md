# GenSecOps AI Security Gateway как услуга

## 1. Какую бизнес-проблему решает

Банк внедряет GenAI-сервисы, но не может считать генератор доверенной точкой принятия решения. Опасный, подменённый или содержащий персональные данные артефакт не должен попадать клиенту, сотруднику или во внешний канал публикации.

GenSecOps AI Security Gateway создаёт независимую точку допуска между генератором и пользователем.

## 2. Почему это важно именно сейчас

- Количество GenAI-сценариев растёт быстрее, чем единые правила их подключения.
- Один и тот же риск возникает у внутренних моделей, внешних API и новых поставщиков.
- Для банка недостаточно получить moderation score: нужен воспроизводимый контроль выдачи, журнал решений и понятная ответственность.

## 3. Где применяется в банке

| Сценарий | Что контролирует Gateway |
|---|---|
| `text2image` | Prompt и результат генерации до выдачи |
| `img2img editing` | Загружаемый исходник и изменённый результат |
| Маркетинговые материалы | Контент до публикации во внешнем канале |
| Клиентские документы | PII, признаки документов и целостность артефакта |
| Внутренние GenAI-сервисы | Единая policy для разных генераторов и подразделений |

## 4. Какие риски снижает

- `PII leakage`: утечка персональных данных через входы и результаты.
- `Fraud documents`: генерация или обработка материалов, похожих на документы.
- `Phishing assets`: подготовка визуальных материалов для социальной инженерии.
- `Deepfake`: риск использования синтетических изображений в мошеннических сценариях.
- `Brand abuse`: несанкционированные или репутационно опасные материалы от имени банка.
- `Regulatory / reputation risk`: отсутствие проверяемого контроля и истории решений.

Текущий MVP демонстрирует enforcement для части этих рисков. Production coverage требует валидированных detector-адаптеров, OCR/DLP и банковской таксономии.

## 5. Почему это нужно купить

- Независимость от генератора: внутренние модели и внешние API проходят одинаковый путь допуска.
- `Fail closed`: сбой safety-компонента не открывает путь к выдаче.
- `Auditable decisions`: решение и технические версии фиксируются в audit.
- `Signed safety passport`: разрешённый артефакт связан с hash, policy version и detector versions.
- `SOC integration readiness`: архитектура предусматривает SIEM forwarding, WORM audit и операционные метрики.

## 6. MVP → Production roadmap

| Этап | Результат |
|---|---|
| MVP | FastAPI gateway, demo UI, deterministic adapters, quarantine, policy, HMAC passport, verified download, append-only JSONL audit |
| Pilot | Валидированные detector-адаптеры, OCR/DLP, банковская таксономия, private object storage, tenant-aware access |
| Production | KMS/HSM signing, WORM audit, SIEM/SOC forwarding, метрики и алерты, review workflow, нагрузочные и adversarial-тесты |
